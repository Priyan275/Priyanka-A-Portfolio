Academic Capstone Project Aug — Oct 2025

Thennilavan Doctor Digital Portal

A role-based clinic management platform built to replace manual, phone-in appointment booking with a real-time, conflict-free scheduling system — designed around how clinics actually run.

The problem

Manual booking meant double-bookings, no live view of doctor availability, and no single source of truth for admins, doctors, or patients.

What I built

  • Three dedicated modules — admin, doctor, patient — each with scoped permissions and views
  • A real-time slot engine that checks live availability before confirming a booking, eliminating double-booking by design
  • Secure login and session handling for all three roles
  • A relational MySQL schema connecting patients, doctors, and appointments with referential integrity

Impact

Reduced appointment booking to a single real-time flow — removing the back-and-forth phone confirmation step and giving every role a live, accurate view of the schedule. (Academic capstone — pilot usage metrics to be added once deployed.)
